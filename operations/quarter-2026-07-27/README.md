## 三条不可突破的边界

1. 公开内容必须区分【事实】【判断】【推测】，保留来源、反方观点和证伪条件。
2. 不提供实时喊单、具体买卖点、收益承诺或个人持仓复制。
3. 真实交易、访谈和付费用户数据只存放在本地 `private/`，不进入公开网站或公开仓库。

## 每周标准节奏

- 周一：定义核心问题；整理公告、财报和产业资料。
- 周二：完成母版事实表；发布小红书1。
- 周三：形成研究结论、反方观点和交易观察清单；撰写长文。
- 周四：发布公众号与官网长文；发布小红书2。
- 周五：完成市场周复盘；发布爱股票社区1。
- 周六：发布小红书3、爱股票社区2和口播；完成访谈或付费交付。
- 周日：半天完全休息；半天填写周复盘并安排下周。

## 交易日固定检查点

- 盘前：只处理公告、原计划和证伪条件。
- 开盘后30分钟：确认是否出现计划内变化。
- 下午收盘前：检查风险与计划，不因无聊增加交易。
- 盘后：记录事实、执行和情绪，研究与交易结论分开保存。

## 每周完成动作

1. 复制 `templates/research-master.md` 到本地私人目录，完成母版事实表。
2. 用 `templates/content-pack.md` 生成各平台版本。
3. 新建官网草稿：`pnpm new:article 文章英文代号`。
4. 发布前运行 `pnpm validate:content`，通过后再把文章状态改为 `published`。
5. 真实交易复制 `templates/trade-decision.md` 到 `private/trades/`。
6. 周日复制 `templates/weekly-review.md` 完成复盘，并更新计分板。
